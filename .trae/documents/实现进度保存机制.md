# 实现进度保存机制

## 1. 需求分析
- 用户可以设定存档名称
- 用户可以读取存档
- 用户可以管理存档（查看、删除、重命名等）

## 2. 实现方案

### 2.1 增强 Player 类
- 修改 `save` 方法，支持自定义存档名称
- 修改 `load` 方法，支持从指定存档加载
- 增加 `get_save_files` 静态方法，获取所有存档文件列表
- 增加 `delete_save` 静态方法，删除指定存档

### 2.2 增强 GameEngine 类
- 修改 `save_game` 方法，支持自定义存档名称
- 修改 `load_player` 方法，支持从指定存档加载
- 增加 `get_save_list` 方法，获取所有存档列表
- 增加 `delete_save` 方法，删除指定存档
- 增加 `rename_save` 方法，重命名存档

### 2.3 增强 CLI 类
- 在主菜单中添加「档案管理」选项
- 实现档案管理界面，包括：
  - 查看所有存档
  - 创建新存档（自定义名称）
  - 加载指定存档
  - 删除存档
  - 重命名存档

### 2.4 更新菜单选项
- 在 `get_menu_options` 中添加档案管理选项

## 3. 具体实现步骤

### 3.1 修改 player.py
- 增强 `save` 方法，支持自定义存档名称
- 增强 `load` 方法，支持从指定存档加载

### 3.2 修改 game_engine.py
- 增强 `save_game` 方法，支持自定义存档名称
- 增强 `load_player` 方法，支持从指定存档加载
- 添加存档管理相关方法

### 3.3 修改 cli.py
- 在主菜单中添加「档案管理」选项

## 4. 预期效果
- 用户可以在主菜单中选择档案管理选项
- 用户可以查看所有可用存档
- 用户可以创建新存档（自定义名称）
- 用户可以加载指定存档
- 用户可以删除存档
- 用户可以重命名存档
